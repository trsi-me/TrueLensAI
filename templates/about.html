About TrueLens AI

TrueLens AI is a locally-running web platform for analysing text and visual content: it estimates the probability that a news article, image, or video is fake or AI-generated versus authentic, and displays a confidence score and a clear decision in English without requiring cloud services at runtime.

Technologies Used

  • Flask: Web framework for serving pages and JSON APIs for detection and history.
  • SQLite: Stores the scan history and statistics in a single file.
  • scikit-learn: TF-IDF pipeline with a machine-learning classifier for the news model.
  • TensorFlow / Keras: EfficientNetB0 with transfer learning for the image model.
  • OpenCV: Extracts frames from video and analyses them with the image model.

Datasets Used

Training relies on two public datasets from Kaggle; files are expected under the datasate/ folder as described in the project guide.

WELFake — News Detection (Text)

  • Source: Kaggle — Fake News Classification (WELFake).
  • Content: News texts with binary labels (Fake / Real).
  • Usage: Train a TF-IDF pipeline with a classifier (e.g. Passive Aggressive) and save the model and vectoriser with joblib.
  • Expected file: WELFake_Dataset.csv inside datasate/.

Can be downloaded manually from the dataset page after logging in, or via the download script if Kaggle API credentials are available.

CIFAKE — Image Detection

  • Source: Kaggle — CIFAKE: Real and AI-Generated Synthetic Images.
  • Content: Real and generated images organised into REAL and FAKE folders.
  • Usage: Train EfficientNetB0 on the training folder (e.g. archive/train) with optional evaluation on archive/test if present.
  • Size: Relatively large; download and extract under datasate/archive.

The folder structure must allow Keras flow_from_directory to recognise both classes.
